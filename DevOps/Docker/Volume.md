# Docker Volumes - Complete Notes

---

# Introduction to Docker Volumes

A **Docker Volume** is a special storage mechanism used to persist data generated by Docker containers.

By default, any data created inside a container is stored in the container's writable layer. When the container is deleted, all data is lost.

Docker Volumes solve this problem by storing data outside the container.

---

# Why Use Docker Volumes?

- Persist data even after the container is removed.
- Share data between multiple containers.
- Store database files safely.
- Backup and restore data easily.
- Better performance than bind mounts.
- Managed directly by Docker.

Docker Volume Commands
1. docker volume create

Creates a new Docker volume.

Syntax
docker volume create <volume_name>
Example
docker volume create mydata
Output
mydata
2. docker volume ls

Lists all Docker volumes.

Syntax
docker volume ls
Example Output
DRIVER    VOLUME NAME
local     mydata
local     mysql_data
3. docker volume inspect

Displays detailed information about a volume.

Syntax
docker volume inspect <volume_name>
Example
docker volume inspect mydata
Sample Output
[
  {
    "Name": "mydata",
    "Driver": "local",
    "Mountpoint": "/var/lib/docker/volumes/mydata/_data"
  }
]
4. docker volume rm

Removes one or more volumes.

Syntax
docker volume rm <volume_name>
Example
docker volume rm mydata

Note: Volume cannot be removed if it is currently in use by a container.

5. docker volume prune

Deletes all unused volumes.

Syntax
docker volume prune
Example
docker volume prune -f

Running Containers with Volumes (-v)

The -v option attaches a volume to a container.

Syntax
docker run -v <volume_name>:<container_path> <image>

Example 1: Nginx HTML Files
docker volume create webdata

docker run -d \
  --name nginx1 \
  -p 8080:80 \
  -v webdata:/usr/share/nginx/html \
  nginx



Use a temporary container to view files.

docker run --rm -it -v mydata:/data alpine sh

Inside container:

cd /data
ls -l


Demonstration: Data Persistence
Step 1: Create Volume
docker volume create demo_volume
Step 2: Start Container
docker run -it --name test1 -v demo_volume:/data ubuntu bash

Inside container:

echo "Hello Docker Volume" > /data/file.txt
exit
Step 3: Remove Container
docker rm test1
Step 4: Start New Container Using Same Volume
docker run -it --name test2 -v demo_volume:/data ubuntu bash

Inside container:

cat /data/file.txt

Output:

Hello Docker Volume

This proves data is persistent.

Sharing Data Between Containers
docker volume create shared_data
Container 1
docker run -it --name writer -v shared_data:/data ubuntu bash

Inside:

echo "Shared file" > /data/test.txt
Container 2
docker run -it --name reader -v shared_data:/data ubuntu bash

Inside:

cat /data/test.txt

Output:

Shared file
Read-Only Volume Mount
docker run -d \
  --name nginx_ro \
  -v webdata:/usr/share/nginx/html:ro \
  nginx
